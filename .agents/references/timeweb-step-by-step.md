# Timeweb Cloud — пошаговый онбординг для Paperclip-деплоя

> Обновлено: 2026-04-22
> Источники: `timeweb.cloud/docs/*`, Playwright-скрины публичных страниц, Terraform registry

## 0. Paperclip в маркетплейсе Timeweb

**Публичный каталог `timeweb.cloud/marketplace/` не содержит готового Paperclip-образа** на момент апр-2026. Там есть: Docker, ispmanager, FASTPANEL, HestiaCP, LAMP, VestaCP, Node.js, GitLab, n8n, Portainer, WordPress, плюс собственный AI-агент Timeweb — **OpenClaw** (это НЕ paperclip.run, а ребрендированный стек Timeweb).

**Но** API `/api/v1/presets/servers/marketplace` может содержать скрытые/новые образы, не отражённые в публичном каталоге. Скрипт `scripts/create_server.py` сначала пробует найти Paperclip через API. Если найден — использует образ, если нет — **fallback на чистую Ubuntu 24.04 + cloud-init с установщиком Paperclip**.

## 1. Регистрация аккаунта

**URL старта:** `https://timeweb.cloud/` → кнопка **«Создать аккаунт»** справа сверху.

**Особенность:** прямая ссылка `https://timeweb.cloud/my/signup` редиректит на `/my/login`. Форма регистрации открывается только с главной.

**Поля:**
- Тип пользователя: **физлицо** или **юрлицо / ИП**
- Название компании (только для юрлиц)
- ФИО (критично — без корректных ФИО невозможно восстановить доступ при потере пароля)
- Email
- Контактный телефон

**После «Зарегистрироваться»:**
- Сразу редирект в ПУ `https://timeweb.cloud/my/`
- **Пароль показывается во всплывающем окне один раз** — на email не дублируется. Сохранить в менеджере паролей либо сразу сменить на свой.
- Email-верификация не блокирует использование

**Опции быстрого входа** (вместо создания с нуля): Passkey, ВК, GitHub, Google — на `/my/login`.

**ФЗ-152:** все дата-центры Timeweb имеют сертификацию до УЗ-1. Для деплоя ИТ-системы, не обрабатывающей ПДн через Paperclip-агентов, дополнительных шагов не требуется.

## 2. Пополнение баланса

**URL:** `https://timeweb.cloud/my/finances` → левое меню **«Баланс и платежи»**.

### Шаги

1. **«Баланс и платежи» → «Оплата»**
2. Выбрать период: **1 / 3 / 6 / 12 мес** (или «Произвольная сумма»)
3. Выбрать способ оплаты
4. Провести платёж — зачисление мгновенное (для большинства способов)

### Способы оплаты

| Способ | Где валиден | Задержка | Для скилла |
|---|---|---|---|
| **Банковская карта МИР / Visa / MC** | РФ и зарубеж (процессинг Uniteller/Сбер) | 3-24ч редко | **Дефолт для физлиц РФ** |
| **СБП** | РФ | Мгновенно | Удобно если карта не проходит |
| **ЮMoney** | РФ | Мгновенно | Альтернатива |
| **SberPay** | РФ | Мгновенно | Альтернатива |
| **Безналичный расчёт** | Юрлица РФ | 1-3 рабочих дня | Для компаний, даёт закрывающие документы |
| **Автопополнение** | После первой оплаты | Мгновенно | Привязать карту → за 7 дней до исчерпания снимется автоматически |

**Скидки при длинной оплате:** 3 мес — 5%, 6 мес — 7%, 12 мес — 10%.

**Зарубежные карты** (Visa/MC не-РФ BIN): работают не у всех эмитентов, явно не описано в доке. Если не прошли — СБП или безнал.

**Криптовалюта** в публичной доке `payment-methods` не упомянута — не закладывать.

**Биллинг-модель:** почасовые списания с баланса **за каждый активный сервер, даже выключенный**. Лицензии / SSL / домены — разово.

### Стоимость Paperclip-сервера (на апр 2026)

Минимальная работоспособная конфигурация Paperclip: **2 vCPU / 4 GB RAM / 50 GB NVMe**.

| Локация | Тариф | Цена/мес | + IPv4 (180 ₽) | Итого/мес |
|---|---|---|---|---|
| **Москва** (MSK-1) | Cloud MSK 50 | 1 062 ₽ | 180 ₽ | **1 242 ₽** |
| Санкт-Петербург (SPB-*) | Cloud SPB 50 | ~1 062 ₽ | 180 ₽ | ~1 242 ₽ |
| Новосибирск (NSK-1) | Cloud NSK 50 | ~1 062 ₽ | 180 ₽ | ~1 242 ₽ |
| **Амстердам** (AMS-1) | Cloud AMS 50 | ~1 242 ₽ | 180 ₽ | ~1 422 ₽ |
| Франкфурт (FRA-1) | Cloud FRA 50 | ~1 242 ₽ | 180 ₽ | ~1 422 ₽ |
| Казахстан (ALA-1) | Cloud ALA 50 | ~1 062 ₽ | 180 ₽ | ~1 242 ₽ |

**Минимальный стартовый депозит для скилла: 1000 ₽ (для AMS-сервера) или 1500 ₽ (для московского).** С запасом на эксперименты (создать/удалить сервер) и IPv4.

**НЕ брать**:
- Cloud MSK 15 (477 ₽, 1 CPU / 1 GB) — недостаточно RAM для Paperclip + Docker
- Cloud MSK 30 (657 ₽, 2 CPU / 2 GB) — на пределе, падает при нагрузке

## 3. API-токен

**URL:** `https://timeweb.cloud/my/api-keys` → раздел **«API и Terraform»** в левом меню.

### Шаги

1. Перейти по ссылке → **«Добавить токен»** (прямой URL: `https://timeweb.cloud/my/api-keys/create`)
2. Заполнить:
   - **Имя токена** — любое читаемое, например `paperclip-deploy`
   - **Срок действия** — **«Бессрочно»**. Иначе скрипты скилла неожиданно начнут падать
   - **«Удалять сервисы по API без кода в Телеграме»** — **ВКЛЮЧИТЬ** (иначе `DELETE /servers/{id}` будет ждать подтверждения в TG-боте, скрипт зависнет)
   - **«Ограниченные права для токена»** — **ВКЛЮЧИТЬ** и выбрать scope (см. ниже)
3. **«Выпустить»** → токен показывается **один раз**, сохранить немедленно
4. Положить в корневой `.env` проекта:
   ```
   TIMEWEB_CLOUD_TOKEN=eyJhbGci...
   ```

**Формат токена:** JWT, начинается с `eyJhbGc`.

### Минимальные scopes для paperclip-deploy

| Сервис | Право | Зачем |
|---|---|---|
| **Облачные серверы** | Полное управление | Create / delete / start / stop серверов |
| **Публичные IP** | Полное управление | Привязка IPv4 при создании |
| **SSH-ключи** | Полное управление | Регистрация твоего ключа на новом сервере |
| **Образы сервера** | Только просмотр | Выбор Ubuntu при create |
| **Проекты** | Только просмотр | Привязка сервера к проекту (опционально) |
| Остальные (БД / S3 / K8s / Load Balancer / DNS / Apps / Dedicated) | **Нет доступа** | Не нужны скиллу, минимизация blast radius при утечке токена |

**Ограничение:** scoped-токены доступны только основному пользователю аккаунта. Доп-пользователи имеют права по своему profile.

## 4. Проверка токена

После вставки в `.env` — прогнать smoke-test:

```bash
source .env && curl -s -H "Authorization: Bearer $TIMEWEB_CLOUD_TOKEN" \
  https://api.timeweb.cloud/api/v1/account/finances | python3 -m json.tool | head -20
```

**Ожидается:** JSON с полями `balance`, `currency`, `hourly_cost`, etc. Если `401 Unauthorized` — токен кривой или истёк.

## 5. Локации — какую выбрать для Paperclip

| Локация | Зоны | Когда брать |
|---|---|---|
| **Москва** MSK-1 | — | Дефолт для РФ-юзеров. **Требует Shadowsocks-прокси** для доступа к Claude/OpenAI (raj geo-block). См. `no-vpn-how-to-get.md` |
| **СПб** SPB-1..4 | 4 зоны | Тот же тариф, что Москва. Если в MSK-1 pool occupancy забит |
| **Новосибирск** NSK-1 | — | Бюджетный fallback, но нет приватных сетей / IPv6 / Load Balancer |
| **Казахстан, Алматы** ALA-1 | — | Если хочешь не-РФ IP без sslocal-прокси. Проверить доступность `api.anthropic.com` с KZ IP отдельно |
| **Амстердам** AMS-1 | — | **Идеально для Paperclip с Claude Max / ChatGPT Plus.** Прямой доступ к Anthropic/OpenAI. Оплата с РФ-карты работает |
| **Франкфурт** FRA-1 | — | То же что AMS-1 + доступен Dedicated CPU |
| ~~Польша~~ | — | **Закрыта** (не предлагать) |

**Дефолт скилла:**
- Если подтверждена подписка ChatGPT Plus / Claude Max → **AMS-1 (Амстердам)**. Не нужен прокси, всё работает из коробки.
- Если РФ-центричный проект и принципиальна MSK-юрисдикция → **MSK-1** + настройка Shadowsocks (из `no-vpn-how-to-get.md`).

## 6. UI мастера создания сервера (8 шагов)

Из `timeweb.cloud/docs/cloud-servers/manage-servers/create-server`. Скилл знает что ждать ровно столько шагов:

1. **Программное обеспечение** — Чистая ОС / Маркетплейс / Мои образы. Для Paperclip:
   - Ubuntu 24.04 (Чистая ОС) — если Paperclip в marketplace отсутствует
   - Docker (Маркетплейс) — если хочется сэкономить 3-5 минут на установке Docker
   - Paperclip (Маркетплейс) — если образ появился к моменту чтения
2. **Регион и зона** — см. раздел 5
3. **Конфигурация** — линейка Premium NVMe, тариф Cloud *SK 50 (2 CPU / 4 GB / 50 GB). Количество серверов — 1
4. **Сеть** — IPv4 (180 ₽), IPv6 (часто бесплатно), приватная сеть (если надо), hostname
5. **Безопасность** — бэкапы (6 ₽/ГБ/мес), DDoS-защита (450 ₽, только СПб/Мск), защита от удаления
6. **Авторизация** — **SSH-ключ (обязательно загрузить свой `~/.ssh/id_ed25519.pub`)**, можно отключить root-пароль
7. **Cloud-init сценарий** — `.sh` или `.yaml`. Сюда скилл кладёт bootstrap Paperclip (установка Docker → pull paperclip-image → systemd unit → proxy env если РФ)
8. **Информация о сервере** — имя (`paperclip-prod`), комментарий, иконка, проект

→ «Заказать» → ~2-5 минут → в «Дашборде» сервера: IP-адрес, root-пароль (если не отключён), команда `ssh ...`.

## 7. Что пользователь ДОЛЖЕН сделать руками

Не автоматизируется скиллом (captcha, 3DS, email-verify и т.д.):

| Шаг | Где | Почему руками |
|---|---|---|
| Регистрация аккаунта | `timeweb.cloud/` → «Создать аккаунт» | Капча + email + ФИО + телефон |
| Пополнение баланса первый раз | ПУ → «Баланс и платежи» | Привязка карты + 3DS-подтверждение в банковском приложении |
| Создание API-токена | ПУ → «API и Terraform» | Форма с scope-селектором, токен показывается 1 раз |
| (Опционально) SSH-ключ | ПУ → «Облачные серверы» → «SSH-ключи» | Можно через API, но быстрее руками загрузить `~/.ssh/id_ed25519.pub` |

**Всё остальное** (создание сервера, управление, удаление, автопополнение по привязанной карте, мониторинг, деплой Paperclip, настройка SS-прокси, OAuth Codex) — автоматизируется скилл-скриптами через `TIMEWEB_CLOUD_TOKEN`.

## 9. Чек-лист для скилла — «пользователь готов»

После прохождения этого файла пользователь должен иметь:

- [ ] Аккаунт Timeweb Cloud (email/пароль сохранены)
- [ ] Баланс ≥ 500 ₽
- [ ] API-токен в `.env` как `TIMEWEB_CLOUD_TOKEN=eyJhbGc...`
