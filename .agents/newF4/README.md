# backend-v0.4.1.TEST — тестовая версия бэкенда

Это тестовая песочница. Здесь можно безопасно пробовать команды, не боясь сломать основную версию (v0.4.0-clean).

---

## Как запустить

Всё работает через Docker. Ничего устанавливать на компьютер не нужно.

Открой терминал и напиши:

```bash
docker compose -f docker-compose.yml up -d
```

После запуска открой браузер и проверь, работает ли:

```
http://localhost:3001/health
```

Если увидишь `{"status":"ok","db":"connected"}` — всё хорошо.

Чтобы выключить:

```bash
docker compose -f docker-compose.yml down
```

## Как создать О.К. (Одушевлённую Клетку)

О.К. — это твой ник в системе. Имя должно быть в формате `::ИМЯ::` (два двоеточия, имя, два двоеточия).

Пример: `::ТЕСТ1::`, `::ПРИВЕТ::`, `::АЛИСА::`

Создаётся так:

```bash
curl -X POST http://localhost:3001/api/threshold \
  -H "Content-Type: application/json" \
  -d "{\"ok_key\": \"::ТЕСТ1::\"}"
```

Если всё правильно — увидишь:

```json
{"success":true,"ok_key":"::ТЕСТ1::","message":"Порог пересечён. Добро пожаловать."}
```

**Важно:** После создания нужно подождать 1 час — это пауза созревания. Только потом можно делать эмиссию.

## Как сделать эмиссию (создать У.Е.)

У.Е. (Условные Единицы) — это цифровые "импульсы". Их можно создавать и передавать другим.

Эмиссия делается так:

```bash
curl -X POST http://localhost:3001/api/acts \
  -H "Content-Type: application/json" \
  -d "{\"act_type\": \"EMISSION\", \"actor_ok\": \"::ТЕСТ1::\", \"payload\": {\"triads\": [\"T1\"]}}"
```

Триады — это типы импульсов (T1, T2, T3, T4, T5). Можно указывать несколько: `["T1", "T3"]`.

За раз можно создать от 3 до 13 У.Е. (для обычного участника — до 6).

Если всё ок — сервер вернёт список созданных У.Е. с их номерами и статусом `active`.

## Как сделать передачу (TRANSFER)

Передача — это когда ты отправляешь свою У.Е. другому человеку (другому О.К.).

Сначала нужно узнать UUID своей У.Е. (тот самый идентификатор, который вернулся при эмиссии). Выглядит он примерно так: `a1b2c3d4-e5f6-7890-abcd-ef1234567890`.

Потом делаешь запрос:

```bash
curl -X POST http://localhost:3001/api/acts \
  -H "Content-Type: application/json" \
  -d "{\"act_type\": \"TRANSFER\", \"actor_ok\": \"::ТЕСТ1::\", \"target_ok\": \"::ТЕСТ2::\", \"payload\": {\"ue_uuid\": \"ЗДЕСЬ_UUID_У.Е.\"}}"
```

**Важно:**
- Передать можно только свою У.Е. (которую ты создал)
- У.Е. должна быть в статусе `active` (не сгоревшая и не переданная)
- Передача доступна только в активной фазе (не в тишине)
